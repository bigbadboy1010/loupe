# Loupe End-to-End Test

## Current deployment baseline

- Public URL: `https://theloupe.team`
- Signaling WebSocket: `wss://signaling.theloupe.team/ws`
- STUN/TURN host: `signaling.theloupe.team:3478`
- TURN external IP from deployment: `212.186.18.125`
- Root path `/` may return `404`; this is expected.
- Required endpoints:
  - `GET /healthz`
  - `WSS /ws`

## Server validation

Run from the Mac before opening Xcode:

```bash
curl -i https://theloupe.team/healthz
npx wscat -c wss://signaling.theloupe.team/ws
nc -vz signaling.theloupe.team 3478
nc -vzu signaling.theloupe.team 3478
```

Expected:

- `/healthz` returns HTTP 200 with JSON status.
- `wscat` connects to `/ws`.
- TCP/UDP 3478 are reachable.

Manual WebSocket protocol smoke test:

```json
{"type":"join","sessionId":"loupe-beta-session","peerId":"manual-test-host","role":"host"}
```

Then:

```json
{"type":"turn-cred"}
```

Expected answer includes one STUN URL and TURN URLs for UDP/TCP.

## Xcode build

Open:

```bash
cd ~/Desktop/Loupe || cd ~/Schreibtisch/Loupe
open Loupe.xcworkspace
```

Build checks:

```bash
xcodebuild \
  -workspace Loupe.xcworkspace \
  -scheme LoupeHost \
  -destination 'platform=macOS' \
  clean build

xcodebuild \
  -workspace Loupe.xcworkspace \
  -scheme LoupeControllerApp \
  -destination 'generic/platform=iOS' \
  CODE_SIGNING_ALLOWED=NO \
  clean build
```

## macOS permissions

LoupeHost requires both permissions:

- System Settings > Privacy & Security > Screen Recording
- System Settings > Privacy & Security > Accessibility

Enable, depending on how the host is launched:

- Xcode
- Terminal
- LoupeHost, if listed

After changing TCC permissions, quit and reopen Xcode.

## Running the iOS controller

Scheme: `LoupeControllerApp`  
Destination: a real iPhone, not the simulator.

Signing:

- Set Apple Developer Team.
- Keep Bundle Identifier: `org.miggu69.loupe.controller`.
- Enable automatic signing.

On the iPhone:

1. Start the app.
2. Tap `QR scannen` and scan the PNG generated by the host.
3. If QR scan is not convenient, paste the printed Pairing Token.
4. Tap `Verbinden`.

Expected controller flow:

- Phase changes from `connecting` to `waitingForHost`.
- TURN/STUN credentials become visible in Diagnostics.
- ICE state changes through `checking` to `connected` or `completed`.
- Video frame counter increases.
- Remote screen appears.
- Touch/drag moves the Mac cursor.

## Diagnostics UI

The iOS app includes:

- Settings screen:
  - Signaling URL
  - Session ID
  - Controller Peer ID
  - Trust Store reset
  - App version/build
- Diagnostics screen:
  - Phase
  - WebSocket/signaling state
  - TURN credential status
  - ICE state
  - PeerConnection state
  - DataChannel state
  - Video frames received
  - Remote video size
  - Last event/error
  - Copy diagnostics to clipboard

The macOS host prints structured `[LoupeHost]` runtime events in the Xcode console.

## Known simulator limitation

The iOS simulator is not considered a valid End-to-End target for Loupe at this stage. It may install and build successfully but can fail to run the QR/camera/WebRTC path reliably. Use a real iPhone for the MVP test.

## Failure analysis order

1. `/healthz` fails: DNS, Caddy, Docker, firewall.
2. `/ws` fails: Caddy WebSocket upgrade or signaling service.
3. `turn-cred` fails: session join, TURN secret, server gating.
4. WebRTC ICE fails: router/NAT/TURN external IP/UDP relay range.
5. No video: Screen Recording permission or ScreenCaptureKit path.
6. Video but no control: Accessibility permission, DataChannel, coordinate mapping.
